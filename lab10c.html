How I Installed Vue.js

To install Vue.js, I decided to include it via a CDN in the <head> section of my HTML file. This approach is simple and doesn't require setting up a complex build environment. Here is the script tag I added:

<script src="https://cdn.jsdelivr.net/npm/vue@2"></script>

By using this method, I was able to easily integrate Vue.js into my project without the need for a package manager like npm.

How I Built the Page

I began by creating a basic HTML structure and including Vue.js via the CDN. In the <body> section, I created a <div> with the id of app, which would serve as the root element for the Vue instance:

<div id="app">...</div>

Inside this div, I used Vue's data binding to display dynamic content. For example, the <h1> tag binds to a data property title in Vue.js:

<h1>{{ title }}</h1>

Here is the Vue.js instance initialization that I placed at the end of the <body> section:

<script>
    new Vue({
        el: '#app',
        data: {
            title: 'Lab10c - Building a Web Page with Vue.js'
        }
    });
</script>

This simple setup allows me to bind the title property to the HTML, dynamically rendering the page content.

Difficulties Encountered

One challenge I faced was that the Vue instance wasn't binding correctly to the DOM elements. After some investigation, I realized the issue was that the Vue script was loading before the DOM was fully parsed. I solved this by moving the Vue instance initialization code to the bottom of the <body> section to ensure that the DOM was ready before Vue.js took over.

Hosting

I hosted this web page on GitHub Pages, which provides free static website hosting. Since Vue.js is a client-side framework, I can use it with a static hosting service like GitHub Pages without needing a backend. GitHub Pages only serves static files, so there was no need to configure a server-side environment or use complex build tools. All I needed was to push my HTML file to a GitHub repository and enable GitHub Pages to serve the content.
