How I Installed Vue.js

I chose Vue.js as my framework for building this web page. To install Vue.js, I included it via a CDN in the <head> section of my HTML file:

<script src="https://cdn.jsdelivr.net/npm/vue@2"></script>

This approach allows me to use Vue.js without setting up a complex build environment or installing additional dependencies.

How I Built the Page

I started by creating a <div> with an id of "app" to serve as the root element for the Vue instance:

<div id="app">...</div>

Inside this div, I used Vue's templating syntax to bind data to the DOM:

<h1>{{ title }}</h1>

Then, at the end of the <body>, I initialized the Vue instance as follows:

<script>
    new Vue({ 
        el: '#app',
        data: {
            title: 'Lab10c - Building a Web Page with Vue.js'
        }
    });
</script>

Difficulties Encountered

Initially, I encountered a problem where the Vue instance was not binding to the DOM elements. This was because the Vue script was loading before the DOM was fully parsed. To fix this, I moved the Vue initialization script to the end of the <body> section.

Another challenge was understanding the reactivity system in Vue.js. After reading the official documentation and experimenting with data binding, I became more comfortable with how Vue.js updates the DOM when data changes.

Hosting

I hosted this web page on Moon's server, which is a shared server environment. Although Moon's server has limitations regarding backend frameworks and some frontend frameworks, Vue.js can be used effectively in a frontend-only setup via CDN. Since Vue.js is a client-side framework, I didn't need any complex server-side setup, making it a great fit for this hosting solution.

The page is available online and accessible through my Moon server hosting, where it runs as a static website without backend dependencies.
